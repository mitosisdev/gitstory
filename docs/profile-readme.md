# GitHub Profile README Integration

Embed an animated gitstory timeline in your GitHub Profile README with auto-regeneration via GitHub Actions.

## Why

Your GitHub profile is where people form first impressions of you as a developer. A commit timeline generated by gitstory turns your history into a visual story — animated, chronological, and shareable. Instead of a static list of pinned repos, visitors see your actual cadence of work: the sprints, the gaps, the long pushes. It shows the developer behind the username.

## Prerequisites

- A GitHub account with a Profile README repo (a repo named `username/username`)
- Node.js 20+ available locally (for a one-time local preview)
- No local install needed for CI — `npx gitstory` handles it automatically

## Step 1: Add the GitHub Actions workflow

Create `.github/workflows/update-timeline.yml` in your Profile README repo with the following content:

```yaml
name: Update gitstory timeline
on:
  push:
    branches: [main]
  schedule:
    - cron: '0 */6 * * *'
jobs:
  update-timeline:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npx gitstory . --output timeline.svg
      - name: Commit timeline
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add timeline.svg
          git diff --staged --quiet || git commit -m "chore: update gitstory timeline [skip ci]"
          git push
```

**What each important part does:**

- **`fetch-depth: 0`** — checks out the full git history, not just the latest commit. Without this, gitstory only sees one commit and the timeline is nearly empty.
- **`schedule: cron '0 */6 * * *'`** — runs every 6 hours automatically, keeping your timeline fresh even when you haven't pushed to the profile repo itself.
- **`git diff --staged --quiet || ...`** — only creates a commit if `timeline.svg` actually changed. If nothing is different, the commit step is skipped, preventing an empty commit loop.
- **`[skip ci]` in the commit message** — tells GitHub Actions not to trigger the workflow again when it pushes the updated SVG. Without this, every auto-commit would kick off another run.

## Step 2: Add the image to your Profile README

In your `README.md`, add:

```markdown
![My commit timeline](https://raw.githubusercontent.com/YOURUSERNAME/YOURUSERNAME/main/timeline.svg)
```

Replace `YOURUSERNAME` with your GitHub username — it appears twice in the URL.

Using the `raw.githubusercontent.com` URL ensures GitHub renders the SVG as an image rather than showing raw XML. Linking directly to the file path in the repo also works, but the raw URL is more reliable across different GitHub UI states.

## Step 3: Push and watch it run

After adding both files, commit and push:

```bash
git add .github/workflows/update-timeline.yml README.md
git commit -m "feat: add gitstory timeline"
git push
```

GitHub Actions will pick up the workflow within a minute. Once it finishes, `timeline.svg` will appear in your repo and the image will render in your profile README automatically.

## Customization

gitstory supports a few flags useful for profile use:

- **`--output <path>`** — controls where the SVG is written. Default is `timeline.svg`. Change it if you want to nest it in a subfolder or use a different name.
- **`--format gif`** — generates an animated GIF instead of an SVG. Useful if you want to embed the timeline somewhere that doesn't support SVG (e.g. npm README badges, some wikis).
- **`--html <path>`** — writes an interactive standalone HTML page alongside the SVG. You can host this on GitHub Pages and link to it from your profile for a richer view:

  ```yaml
  - run: npx gitstory . --output timeline.svg --html timeline.html
  ```

  Then link it from your README:

  ```markdown
  [View interactive timeline](https://YOURUSERNAME.github.io/YOURUSERNAME/timeline.html)
  ```
